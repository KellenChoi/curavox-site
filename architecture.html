Voice Session Flow

1

Record

User taps the 200px green button. Audio recorded in mono 16kHz WAV via expo-audio.

Device Microphone
2

Transcribe (STT)

Audio base64 sent to transcribe/ Edge Function → Deepgram Nova-2. Korean-first strategy with keyword boosting (call:2, text:2, search:1.5).

Deepgram Edge Function
3

Think (LLM)

Transcript + system prompt + memories + health + mood + history sent to conversation/ Edge Function → Claude. Smart routing: Sonnet for actions, Haiku for simple chat. May use web_search tool. Prompt caching saves ~90% on static instructions.

Claude Sonnet/Haiku Edge Function
4

Speak (TTS)

Reply text sanitized (units, markdown, emoji stripped) → speak/ Edge Function → ElevenLabs streaming audio. Played via expo-audio.

ElevenLabs Edge Function
5

Execute Action Tags

Client parses 16+ tag types from response. Calls, texts, reminders, deeplinks, app opens, contact saves — all handled on-device after speaking.

On-Device
6

Track Analytics

Voice session event + action tags + deeplink success/fail + LLM token usage saved to analytics_events table. Fire-and-forget, never blocks UX.

Supabase

System Prompt Structure (Input to Claude)

Static Instructions ~15,000 tokens (CACHED 5min, 90% savings)
Hana persona, conversation rules, action tag formats, scam education, booking rules, contact handling, locale routing, emergency protocols
Dynamic Context ~1,000-2,000 tokens (per-request)
User memories (7 categories) • Health profile (meds, conditions, allergies) • Mood history (14 entries) • Conversation digests (7 days) • Pattern insights • Family members • Proactive messages • Weather • Current time (DST-aware) • User locale + GPS
Conversation History ~500-2,000 tokens (last 10 turns)
User and assistant messages from current session. Resets on app restart. Seeded from Supabase on first session.

Action Tags (16 types)

[CALL:] Phone call — searches phone contacts, shows picker
[TEXT:] SMS — searches contacts, pre-fills message
[EMAIL:] Email — opens mailto: with subject/body
[KAKAO:] KakaoTalk — Korean messaging app
[OPEN:] Direct URL — opens specific webpage/app
[YOUTUBE:] YouTube — opens app search
[NAVER:] Web search — Naver (KR) or Google (US)
[SHOP:] Shopping — Coupang/Amazon/delivery apps
[MAP:] Maps — Naver/Google/Apple Maps
[APP:] Open app — 30+ apps with fallbacks
[REMINDER:] Set reminder — saves to DB + notification
[MEMORY:] Save memory — 7 categories, permanent
[SAVE_CONTACT:] Save to phone book
[HEALTH:] Health data — meds, conditions, symptoms
[MOOD:] Mood score — 1-5 per conversation
[FAMILY:] Register family member for dashboard

Backend Architecture

🗄️ Supabase PostgreSQL

  • profiles — name, language, timezone, push token
  • conversations — full chat history
  • memories — 7 categories, permanent
  • reminders — title, time, recurrence
  • health_profile — meds, conditions, allergies
  • contacts — emergency only (legacy)
  • mood_log — 1-5 score per conversation
  • analytics_events — voice, deeplinks, tokens, cost
  • family_members + family_alerts
  • conversation_digests — daily summaries
  • pattern_insights — health trends
  • proactive_messages — morning/afternoon/evening
  • app_config — version check, update URLs

⚡ Edge Functions (10)

  • conversation — main Claude proxy + tag parsing
  • transcribe — Deepgram STT proxy
  • speak — ElevenLabs TTS proxy
  • translate-image — Claude Vision for camera
  • speak-demo — unauthenticated demo TTS
  • daily-digest — CRON: daily conversation summary
  • analyze-patterns — CRON: health trends + alerts
  • proactive-context — CRON: 3x daily push messages
  • send-family-alerts — CRON: email family (WIP)
  • _shared/cron-auth.ts — service role key guard
